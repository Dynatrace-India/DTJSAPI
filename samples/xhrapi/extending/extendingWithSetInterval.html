Extending action times when using setInterval

Agentconfig
Needed agent module configuration:
Support for XmlHttpRequest

There are many ways multiple XHRs could be intended to be grouped together, even if their implementation suggests otherwise. The following code shows how the dtrum api can be leveraged to create a link between loosely related XHRs using setInterval.

enterXhrAction

leaveXhrAction

This code tells the JavaScript Agent to create an XHR action in case a user input or an action is currently active. Clicking the Load Data button in the example results in the User Action shown in the following figure. The resulting XHR action will contain both requests to ../../ajax.txt, even though the second one is fired later within a setInterval callback.

Code snippet
function loadData(url, callback) { var xhr = new XMLHttpRequest(); xhr.open('GET', url); xhr.onreadystatechange = function() { if (xhr.readyState === 4) { callback(xhr.responseText); } }; xhr.send(null); } function closeAction(actionId) { loadData('../../ajax.txt?' + new Date().getTime(), function (data2) { document.getElementById('resultDiv2').innerHTML = 'Second XHR result:' + data2; dtrum.leaveXhrAction(actionId); }); } function init() { var url = '../../ajax.txt?' + new Date().getTime(); loadData(url, function(data) { //adding the timestamp to prevent caching document.getElementById('resultDiv').innerHTML = 'First XHR result:' + data; var actionId = dtrum.enterXhrAction("arbitrary", 3, url); var numIntervals = 2; var intId = setInterval(function () { if (--numIntervals === 0) { clearInterval(intId); closeAction(actionId); } }, 100); }); }
